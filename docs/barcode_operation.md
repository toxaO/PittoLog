# バーコード運用

## バーコード形式

Code128 の PNG を発行します。LS2208 は 1D バーコードリーダーなので、QR ではなく Code128 を初期方針にします。

## 接頭辞

- `ITEM:*`: 物品
- `DEPT:*`: 部署
- `ACTION:001`: 確認
- `ACTION:002`: キャンセル

スキャナの設定によって `:` が `+` として入力される場合があります。この場合も、アプリ側で `ACTION+001` を `ACTION:001` として扱います。

## 貸出

1. `ITEM:*` を読む
2. `DEPT:*` を読む
3. `ACTION:001` を読む

## 返却

1. `ITEM:*` を読む
2. `ACTION:001` を読む

## 注意

バーコードリーダーは Enter 付きのキーボード入力として扱います。アプリの読み取り欄にフォーカスがある状態で使います。

## 登録

管理タブから登録します。

- カテゴリ登録: 物品登録の前にカテゴリを作成します。
- 物品登録: 物品名とカテゴリを選択すると、`ITEM:000001` から自動採番します。
- 物品登録の手動バーコード指定: 必要な場合だけオプションで指定します。`ITEM:000001` または `000001` のどちらでも登録できます。
- 部署登録: 部署名を入力すると、`DEPT:0001` から自動採番します。
- 部署登録の手動バーコード指定: 必要な場合だけオプションで指定します。`DEPT:0001` または `0001` のどちらでも登録できます。
- スキャナ設定によって `ITEM:000001` が `ITEM+000001` と入力される場合があります。この場合もアプリ側で `ITEM:000001` として扱います。

動作確認用バーコードをそのまま登録する場合は、物品名に `物品1`、カテゴリに `test` のような任意の値を入れ、手動バーコード指定を有効にしてください。

## CSV物品読込

管理タブの物品画面から CSV を読み込めます。登録できるのは物品のみです。

対応する列名:

- `物品名` または `name`: 必須
- `カテゴリ` または `category`: 任意。未登録カテゴリは自動作成します。
- `バーコード` または `barcode`: 任意。空なら自動採番します。

例:

```csv
物品名,カテゴリ,バーコード
プロジェクター,AV機器,
スクリーン,AV機器,ITEM:000010
机,備品,
```

動作テスト用バーコードに対応する読込サンプルは [assets/test_items_import.csv](assets/test_items_import.csv) です。

## 動作テスト用シート

[assets/pittolog_test_barcodes_a4.pdf](assets/pittolog_test_barcodes_a4.pdf) を A4 サイズ、印刷倍率 100% で印刷してください。

動作テスト用シートとアプリの「バーコード発行」タブで作成するバーコードは、末尾に Enter を含めます。画面上は `ITEM:000001 + Enter` のように表示します。

印刷後に読み取りづらい場合は、線幅、高さ、余白を調整した PNG を発行して確認します。

## 任意バーコードのA4出力

管理タブの「バーコード発行」で、任意のバーコード一覧をA4形式のPDFとして出力できます。1ページに収まらない件数を指定した場合は、複数ページのPDFとして出力します。

物品バーコードを発行する場合は、物品IDを指定して発行リストへ追加できます。

```text
1, 3 5-7
```

また、管理タブの物品一覧で行を選択して「発行リストへ追加」を押すと、その物品がバーコード発行タブの一覧へ追加されます。

入力形式:

```text
物品1,ITEM:000001
物品2,ITEM:000002
確認,ACTION:001
キャンセル,ACTION:002
```

カンマの代わりにタブ区切りも使えます。区切りがない行は、同じ文字列をラベルとバーコード値の両方に使います。

## 登録内容の変更

管理タブで以下を変更できます。

- 物品: 物品名、カテゴリ、必要な場合のみバーコード
- カテゴリ: カテゴリ名
